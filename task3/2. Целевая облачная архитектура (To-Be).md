### Принципы:

Убираем локальные сервера (Windows, SQL) – их заменяет лёгкий Edge Gateway (может работать на Raspberry Pi или старом планшете) + облачный бэкенд (SaaS/PaaS).

Касса становится тонким клиентом (Web‑приложение на планшете Android/iPad). Планшет не имеет драйверов для железа.

Для работы с разнородным оборудованием вводим Hardware Abstraction Layer (HAL) / Local Edge Gateway, который:

Принимает команды от облачного POS через защищённое WebSocket‑соединение.

Управляет фискальным регистратором и банковским терминалом через стандартные драйверы (универсальные или адаптированные).

Предоставляет единый API для облака: openDrawer, printReceipt, processPayment(amount, method).

__Офлайн-режим:__ Edge Gateway кэширует входящие чеки и события, а планшет может работать в автономном режиме (базовая продажа наличных). После восстановления интернета данные синхронизируются с облаком.
