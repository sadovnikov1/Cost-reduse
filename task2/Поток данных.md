__Поток данных при автоматическом маппинге товаров (на примере SKU)__

### 2.1. Исходные данные (из старой кассы купленного магазина)
Пример строки из CSV-файла, выгруженного из старой кассы:

Наименование (old_name)	Штрихкод (barcode)	Производитель	Цена
Шамп. Pantene 200 Pro-V	4601234567890	Pantene	199.00
Проблемы:

* Аббревиатура «Шамп.» (вместо «Шампунь»).
* Отсутствует объём «200 мл».
* Нет артикула производителя.
* Может не быть штрихкода или он не совпадает с базой VitaStore.

### 2.2. Эталонная модель товара в SAP ERP (целевая информационная модель)
VitaStore хранит товары в SAP с набором атрибутов:

json
{
  "internal_id": "VITA_123456",
  "full_name": "Шампунь Pantene Pro-V 200 мл",
  "brand": "Pantene",
  "category": "Уход за волосами",
  "volume_ml": 200,
  "barcode": "4601234567890",
  "supplier_article": "PTN-200-PROV",
  "price_standard": 199.00
}
### 2.3. Этапы автоматического маппинга (AI pipeline)
### Шаг 1. Нормализация входных данных

Сырая строка «Шамп. Pantene 200 Pro-V» проходит через препроцессор:

Очистка от спецсимволов, лишних пробелов.

Расширение аббревиатур (словарь: «Шамп.» → «Шампунь»).

Выделение марки (Pantene), объёма (200), серии (Pro-V).

### Результат нормализации:

json
{
  "normalized_name": "Шампунь Pantene 200 Pro-V",
  "extracted_brand": "Pantene",
  "extracted_volume": 200,
  "extracted_line": "Pro-V"
}
### Шаг 2. Поиск кандидатов в эталонной базе (SAP)

AI-сервис выполняет несколько видов поиска:

Точный поиск по штрихкоду (если есть) – даёт самый быстрый результат.

Поиск по бренду + объёму – для получения ограниченного набора кандидатов.

Нечёткое сравнение строк (алгоритмы Levenshtein, Jaro-Winkler) для нормализованного названия.

Допустим, штрихкод отсутствует или не найден. Тогда AI ищет по бренду «Pantene» и объёму «200» и получает 3 кандидата:

ID	Полное название	Объём	Серия	Цена	Совпадение
1	Шампунь Pantene Pro-V 200 мл	200	Pro-V	199	98%
2	Шампунь Pantene Classic 200 мл	200	Classic	185	65%
3	Кондиционер Pantene Pro-V 200 мл	200	Pro-V	199	45%
### Шаг 3. Вычисление confidence score

Модель машинного обучения (логистическая регрессия или градиентный бустинг) рассчитывает уверенность для каждого кандидата на основе:

Совпадения бренда (высокий вес).

Совпадения объёма (высокий вес).

Совпадения ключевых слов (Pro-V, «шампунь» vs «кондиционер»).

Исторической частоты правильных маппингов для похожих названий.

В примере кандидат 1 получает confidence = 98%, кандидат 2 – 65%, кандидат 3 – 45%.

### Шаг 4. Принятие решения

Если confidence ≥ 95% (как в примере) – система автоматически принимает кандидата 1, фиксирует маппинг и передаёт в SAP.

Если confidence < 95% – система помечает позицию «на верификацию» и отправляет товароведу в виде таблицы с предложенными вариантами.

### Шаг 5. Формирование целевого формата для SAP

После выбора (автоматически или вручную) AI-сервис формирует запись для загрузки в SAP

### Шаг 6. Обучение модели (feedback loop)

После того как товаровед подтвердил или исправил маппинг (даже для уверенности <95%), этот пример (source → target) добавляется в обучающую выборку. 
[Синхронизация с магазинами] → новая касса получает готовый справочник
### 2.5. Ожидаемый результат для SKU из примера
Время обработки одной позиции AI: менее 100 мс.
Товаровед не участвует в этом конкретном случае.
Система выдаёт итоговый прайс-лист для нового магазина, где «Шамп. Pantene 200 Pro-V» уже привязан к внутреннему ID VITA_123456 с корректной ценой и категорией.
