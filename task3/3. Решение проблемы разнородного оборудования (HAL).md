### Как работает Hardware Abstraction Layer:

При первом включении магазина Edge Gateway определяет тип фискального регистратора и банковского терминала (можно вручную ввести модель через веб-интерфейс).

Gateway загружает соответствующий драйвер из облачного репозитория (или имеет универсальный драйвер с настройками протокола).

Cloud POS отправляет команду print_receipt(receipt_data) на Gateway. Gateway преобразует её в команды конкретного устройства (например, ESC/POS для принтера или XML для fiscal printer).

Для банковского терминала: Cloud POS вызывает process_payment(amount, payment_method). Gateway связывается с терминалом через локальный протокол (например, ARCUS-2 для Сбера, или TCP/IP для Ingenico) и возвращает статус (approved/declined).

### Преимущества:

Облачный POS не зависит от модели оборудования.

При смене терминала (например, после покупки новой сети) меняется только конфигурация Gateway, без обновления кассового ПО.

Можно использовать разные типы терминалов в разных магазинах, но единый облачный интерфейс.
